\# Coding Standards



\## Purpose



Define coding conventions used throughout the project.



\---



\## Principles



\- Readability over cleverness

\- Business-first implementation

\- Single Responsibility Principle

\- Keep functions small

\- Prefer composition over inheritance

\- Avoid duplicated logic



\---



\## Naming



\### Classes



PascalCase



\### Functions



camelCase



\### Variables



camelCase



\### Constants



UPPER\_SNAKE\_CASE



\### Files



kebab-case



\---



\## Comments



Explain why.



Avoid explaining obvious code.



\---



\## Formatting



\- Consistent indentation

\- One responsibility per function

\- No dead code
