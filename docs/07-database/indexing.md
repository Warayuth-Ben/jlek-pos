\# Indexing



\## Purpose



Defines indexing strategies for database performance.



\---



\## Objectives



Indexes should:



\- Improve query performance.

\- Reduce lookup time.

\- Support sorting.

\- Support filtering.



\---



\## Index Types



\- Primary Index

\- Unique Index

\- Composite Index

\- Secondary Index



\---



\## Index Strategy



Create indexes for:



\- Primary Keys

\- Foreign Keys

\- Frequently searched columns

\- Frequently sorted columns



\---



\## Composite Indexes



Composite indexes should follow query patterns.



Avoid unnecessary composite indexes.



\---



\## Maintenance



Indexes should be reviewed periodically.



Unused indexes should be removed.



\---



\## Principles



\- Optimize common queries.

\- Avoid excessive indexes.

\- Balance read and write performance.
