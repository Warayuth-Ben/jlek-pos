\# Table Assignment



\## Purpose



Defines how dine-in orders are associated with restaurant tables.



\---



\## Business Rules



\### 1. Every dine-in order must be assigned to one table.



Table assignment occurs when the order is created.



\---



\### 2. Multiple orders may belong to the same table.



Additional customer requests become part of the same dining session.



\---



\### 3. A table may not be assigned to multiple independent dining sessions simultaneously.



All active orders on the same table belong to one business session.



\---



\### 4. Table assignment becomes part of the business history.



Changes to table assignment must remain traceable.
