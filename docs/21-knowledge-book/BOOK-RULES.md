# Book Rules

1. หนังสือไม่สร้าง Requirement ใหม่

2. หนังสือไม่เปลี่ยน Business Rule

3. หนังสืออ้างอิง Specification เสมอ

4. ทุกบทเริ่มจากเหตุการณ์จริง

5. ทุกบทมีสามมุมมอง

- ร้าน
- ระบบ
- สถาปนิก

6. ทุกบทต้องตอบคำถาม

- ร้านกำลังทำอะไร
- ระบบกำลังทำอะไร
- ทำไมจึงออกแบบเช่นนี้

7. หนังสือต้องอ่านสนุก

8. ผู้อ่านไม่จำเป็นต้องมีพื้นฐานด้าน Software

9. เน้น "Why" มากกว่า "How"

10. เป้าหมายสูงสุดคือการรักษาองค์ความรู้ของร้าน
