\# Rule Writing Guide



\## Purpose



Defines how Business Rules should be written.



\---



\## Characteristics



A Business Rule should be:



\- Atomic

\- Testable

\- Unambiguous

\- Independent



\---



\## Recommended Structure



Rule ID



Title



Description



Rationale



Exceptions



Related Documents



\---



\## Example



Rule:



An Order must contain at least one Order Item before confirmation.



Reason:



Prevent invalid orders from entering the kitchen workflow.
