\# Writing Style



\## General Style



Documentation should use:



\- Clear language

\- Active voice

\- Short paragraphs

\- Consistent terminology



\---



\## Avoid



\- Ambiguous wording

\- Personal opinions

\- Implementation-specific explanations

\- Marketing language



\---



\## Preferred Language



Describe facts rather than assumptions.



Example:



Good:



```

The Order is confirmed after validation succeeds.

```



Avoid:



```

The system probably confirms the order.

```



\---



\## Consistency



Use the same wording throughout the project.



Example:



Always use:



\- Order

\- Bill

\- Kitchen Ticket

\- Payment



Avoid using synonyms interchangeably.
