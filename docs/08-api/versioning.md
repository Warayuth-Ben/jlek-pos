\# API Versioning



\## Purpose



Defines API version management.



\---



\## Version Strategy



APIs should use explicit versioning.



Example



/api/v1/



\---



\## Compatibility



New versions should preserve backward compatibility whenever practical.



\---



\## Deprecation



Deprecated APIs should:



\- Be documented.

\- Remain available during the deprecation period.

\- Provide migration guidance.



\---



\## Principles



\- Stable public contracts.

\- Predictable upgrades.

\- Controlled evolution.
