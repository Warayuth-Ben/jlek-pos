# Purpose

เอกสารนี้กำหนดหลักการกลางสำหรับการออกแบบ UI ของ JLek Platform

UI ทุกหน้าจะต้องอ้างอิงหลักการนี้ก่อนเริ่มออกแบบหรือพัฒนา

------------------------------------------------------------------------

# Mission

เป้าหมายของ UI ไม่ใช่การสร้างหน้าจอที่สวยที่สุด

แต่คือการทำให้การทำงานของร้านง่ายขึ้น เร็วขึ้น และผิดพลาดน้อยลง

------------------------------------------------------------------------

# Core Principles

## 1. Business Before Interface

UI ต้องสะท้อน Workflow ของธุรกิจ ไม่ใช่บังคับให้ธุรกิจเปลี่ยนตาม UI

------------------------------------------------------------------------

## 2. User Thinks in Tasks

ผู้ใช้คิดเป็น "งาน"

ไม่ใช่ "หน้า"

ดังนั้นระบบจะออกแบบเป็น Workspace ก่อน Screen

------------------------------------------------------------------------

## 3. Workspace Before Page

ทุกหน้าต้องเป็นส่วนหนึ่งของ Workspace

ไม่สร้างหน้าที่ไม่มีบริบทของการทำงาน

------------------------------------------------------------------------

## 4. Workflow Before Layout

เริ่มจากลำดับการทำงาน

แล้วจึงออกแบบ Layout

------------------------------------------------------------------------

## 5. Progressive Disclosure

แสดงเฉพาะข้อมูลที่จำเป็นในแต่ละช่วง

ลดภาระในการตัดสินใจของผู้ใช้

------------------------------------------------------------------------

## 6. Feedback Always

ทุก Action ต้องมี Feedback

-   Loading
-   Success
-   Warning
-   Error
-   Printing
-   Saving

------------------------------------------------------------------------

## 7. Keyboard First

ระบบต้องรองรับการทำงานด้วย Keyboard อย่างมีประสิทธิภาพ

Mouse เป็นตัวช่วย ไม่ใช่ข้อบังคับ

------------------------------------------------------------------------

## 8. Consistency

Component เดียวกัน

ต้องมีพฤติกรรมเดียวกันทั่วทั้งระบบ

------------------------------------------------------------------------

## 9. Errors Must Teach

Error Message ต้องอธิบาย

-   เกิดอะไรขึ้น
-   เพราะอะไร
-   ควรทำอย่างไรต่อ

------------------------------------------------------------------------

## 10. Safe by Default

การกระทำที่ย้อนกลับไม่ได้

ต้องมีการยืนยันเสมอ

------------------------------------------------------------------------

# UI Success Metrics

-   ผู้ใช้ใหม่เริ่มใช้งานได้รวดเร็ว
-   ลดจำนวนการคลิกที่ไม่จำเป็น
-   ลดข้อผิดพลาดในการใช้งาน
-   รองรับการทำงานช่วงเร่งด่วนของร้านอาหาร
-   UI ซ่อนความซับซ้อนของระบบไว้เบื้องหลัง

------------------------------------------------------------------------

# North Star

> ออกแบบ UI เพื่อช่วยให้คนทำงาน ไม่ใช่เพื่อโชว์ความสามารถของระบบ

> เมื่อผู้ใช้ลืมไปว่ากำลังใช้โปรแกรม และจดจ่อกับงานของตัวเอง นั่นคือ UI ที่ประสบความสำเร็จ
